### الـ Onboarding (3 شاشات سريعة)

**الشاشة 1 - "اكتشف متاجر المنيا"**
```

│      [ تخطي ]                 │

```

**الشاشة 2 - "قارن الأسعار بسهولة"**
- شوف أسعار كل المتاجر في مكان واحد
- قارن نفس المنتج في أكتر من متجر
- وفّر وقتك وفلوسك

**الشاشة 3 - "احجز واستلم"**
- احجز المنتج أونلاين
- روح المتجر وجرّب قبل ما تشتري
- لو عجبك ادفع واستلم... لو لأ مفيش أي إلزام!

**بعد الـ Onboarding:**
- العميل يدخل **الصفحة الرئيسية مباشرة** بدون تسجيل
- يقدر يتصفح كل المتاجر والمنتجات والأسعار **بدون حساب**
- التسجيل مطلوب **فقط** عند الحجز أو إضافة للـ Wishlist

### مراحل الطلب (Lifecycle):

```
reserved → confirmed → ready → completed
   ↓                              ↑
   ↓         (أو)                 │
expired     cancelled          العميل استلم ودفع
(48 ساعة)   (العميل ألغى)
```

**1. reserved (محجوز):**
- العميل حجز المنتج
- التاجر استلم notification
- عداد 48 ساعة شغال

**2. confirmed (مؤكد من التاجر):**
- التاجر أكد إن المنتج متوفر فعلاً
- (اختياري: ممكن نتخطاها ونروح ready مباشرة)

**3. ready (جاهز للاستلام):**
- التاجر جهّز المنتجات
- العميل يستلم notification: "طلبك جاهز! 🎉"

**4. completed (مكتمل):**
- العميل راح المتجر
- جرّب المنتج
- عجبه → دفع نقداً → التاجر أدخل الـ Pickup Code
- الطلب اتقفل ✅
- العمولة (3%) تتسجل

**5. expired (منتهي):**
- الـ 48 ساعة خلصوا والعميل مجاش
- المنتج يرجع متاح تاني

**6. cancelled (ملغي):**
- العميل ألغى الحجز قبل ما يروح
- المنتج يرجع متاح

### التقييم:
- بعد ما الطلب يبقى `completed`
- يوصل notification للعميل: "قيّم تجربتك في ستايل هب! ⭐"

  - المتجر (1-5 نجوم + تعليق)

- يوصله notification لو المنتج نزل عليه خصم
- يوصله notification لو المنتج قرب يخلص

### كيف تشتغل العمولة؟

```
عميل يطلب منتج بـ 1000 ج.م
         ↓
العميل يستلم ويدفع 1000 ج.م للتاجر نقداً
         ↓
الطلب يبقى completed
         ↓
تسجيل عمولة 30 ج.م (3%) في جدول commissions
         ↓
الأدمن يحصّل العمولة دورياً (أسبوعياً/شهرياً)
         ↓
عن طريق: فودافون كاش / تحويل بنكي / نقداً
         ↓
الأدمن يسجل "تم التحصيل" في النظام
```

### جدول `commissions` المقترح:
```sql
commissions (
  id UUID PRIMARY KEY,
  order_id UUID REFERENCES orders(id),
  store_id UUID REFERENCES stores(id),
  order_amount DECIMAL,        -- المبلغ الكلي للطلب
  commission_rate DECIMAL,     -- 0.03 (3%)
  commission_amount DECIMAL,   -- المبلغ المحسوب
  status TEXT DEFAULT 'pending', -- pending / collected
  collected_at TIMESTAMP,
  collected_method TEXT,       -- vodafone_cash / bank / cash
  notes TEXT,
  created_at TIMESTAMP DEFAULT now()
)
```

# 📋 الفرق عن النظام القديم (بدون اشتراكات)

| البند | النظام القديم | النظام الجديد |
|-------|--------------|---------------|
| **دخول التاجر** | يدفع 1000-2000 ج.م مقدماً | **مجاني 100%** |
| **مصدر الربح** | اشتراكات شهرية | **عمولة 3% على البيع** |
| **الباقات** | 3 باقات مختلفة | **لا يوجد - كل التجار متساويين** |
| **حاجز الدخول** | عالي (التاجر يتردد) | **صفر (مفيش مخاطرة)** |
| **رسالة التسويق** | "اشترك في المنصة" | **"سجل مجاناً وادفع بس لما تبيع"** |
| **جداول الداتابيز** | subscriptions + plans | **commissions** |
| **تقييد الميزات** | حسب الباقة | **كل الميزات مفتوحة للكل** |
| **التحصيل** | قبل الخدمة (مقدماً) | **بعد البيع (دوري)** |
| **أولوية الظهور** | حسب الباقة (Premium أولاً) | **حسب التقييم والنشاط** |

# ✅ مميزات النموذج الجديد

1. **أسهل في تجنيد التجار** - "مفيش ما تدفعه مقدماً"
2. **ثقة أكبر** - "ادفع بس لما تبيع فعلاً"
3. **عدالة** - كل التجار ليهم نفس الميزات
4. **مرونة** - لو التاجر مش باع مش هيتحمل خسارة
5. **تحفيز** - Outfit Hub مصلحته إن التاجر يبيع أكتر
